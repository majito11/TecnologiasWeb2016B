Promises (goodbye callback hell)

Promesas nos permiten dirigir nuestra código horizontal (demonios de devolución de llamada): func1(function (value1) { func2(value1, function (value2) { func3(value2, function (value3) { func4(value3, function (value4) { func5(value4, function (value5) { // Do something with value 5 }); }); }); }); }); En código vertical: func1(value1) .then(func2) .then(func3) .then(func4) .then(func5, value5 => { // Do something with value 5 });

Descripcion

La interfaz Promise representa un proxy para un valor no necesariamente conocido cuando se crea la promesa. Permite asociar manejadores a un eventual éxito o fracaso de acciones asíncronas. Esto permite que los métodos asíncronos devuelvan valores como métodos sincrónicos: en lugar del valor final, el método asíncrono devuelve una promesa de tener un valor en algún momento en el futuro. Una promesa pendiente puede llegar a ser cumplida con un valor o rechazada con una razón. Cuando cualquiera de ellas sucede, los manejadores asociados encolados se llaman mediante el método then de la promesa. (Si la promesa ya se ha cumplido o rechazado cuando se conecta un controlador correspondiente, el manejador será llamado, así que no hay condición de carrera entre el final de una operación asincrónica y sus manejadores se adjuntan).

Beneficios de Promesas : Manejo de errores utilizando un montón de devoluciones de llamada anidados pueden obtener caótica. El uso de promesas, tenemos un camino claro a burbujear errores y manejarlos adecuadamente. Por otra parte, el valor de una promesa después de que se haya resuelto / rechazado es inmutable - que nunca va a cambiar.
